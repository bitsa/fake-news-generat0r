## What this bundle gives you

A hi-fi React prototype of the frontend. Use it to match the prototype's UX,
structure, component names, layout, and copy when porting to the real
[`frontend/`](../../frontend/).

- **Visuals + interactions:** `Fake News v2 (Locked).html` (open in a browser).
- **Components to port:** `src-locked/*.jsx` → TypeScript components under
  [`frontend/src/components/`](../../frontend/src/components/) (PascalCase per ADR-14).
- **Design tokens:** `tokens.css` — extracted from the locked HTML's `<style>`
  block. Lift values into `tailwind.config.js theme.extend` (Tailwind v3, ADR-13).
  Do not invent new colors.
- **Mock data shapes:** `src/data.jsx` shows the field names and value examples
  the prototype renders against. Treat this as illustrative only — the real
  shapes are defined in [`contracts.md`](../../contracts.md) and may differ.

## Locked variant — already decided

The locked HTML and `src-locked/` represent the chosen variant. Do not
re-litigate these in implementation:

- Layout: `split`
- Fake-treatment loudness: `medium`
- Source branding: `dot`

The explorable `Fake News.html` + `src/` + `tweaks-panel.jsx` are kept for
reference only — they show the alternatives that were considered. Port from
`src-locked/`, not `src/`.

## Porting rules

1. **Lift copy literally.** Headings, button labels, empty states, error
   strings, microcopy — copy from the prototype JSX into the real components
   verbatim. Do not paraphrase.
2. **Match component names and prop shapes.** Names in `src-locked/*.jsx` map
   1:1 to the components in [`frontend/src/components/`](../../frontend/src/components/).
   Prop interfaces follow `{ComponentName}Props` per [`conventions.md`](../../conventions.md).
3. **Translate types from contracts, not from the prototype.** The JSX uses
   loose JS shapes; the real TypeScript types live in
   [`contracts.md`](../../contracts.md) under "TypeScript Types" and should be
   mirrored exactly in `frontend/src/types/api.ts`.
4. **Tailwind utilities only.** No inline `style` props except for
   JS-driven dynamic values (progress bar widths, streamed text, etc).
5. **Server state via React Query.** Do not hold server data in `useState`
   (per ADR-7).

## Components in the design that aren't in the MVP API contract

Some components in the JSX render features the API contract doesn't cover yet:

- `InlineDiff` — token-level diff between original and fake.
- `EntitiesCard` — extracted people / orgs / places.
- The admin "scrape runs" / progress modal (`scrape_runs` table, run polling).
- Chat structured renders (entity / diff messages with `kind` + `payload`).

Treat these as **design reference only**. Do not invent backend endpoints or
schema for them. If one of these features is pulled into scope, it will land
via a real spec change — new ADR in [`decisions.md`](../../decisions.md) and a
contract update in [`contracts.md`](../../contracts.md) — before any code is
written.
