This bundle is the **visual / interaction design reference** for the Fakeline
frontend. Use it for porting components, copy, microcopy, layout, and design
tokens — not for backend / infra decisions.

## How to use this bundle

1. Open [`Fake News v2 (Locked).html`](./Fake%20News%20v2%20(Locked).html) in a
   browser. This is the visual + interaction reference.
2. Read [`CLAUDE.md`](./CLAUDE.md) for the porting rules.
3. Read the root specs ([`architecture.md`](../../architecture.md),
   [`contracts.md`](../../contracts.md), [`conventions.md`](../../conventions.md),
   [`decisions.md`](../../decisions.md)) for anything backend / infra / contract.
4. Port `src-locked/*.jsx` → TypeScript components under
   [`frontend/src/components/`](../../frontend/src/components/) (PascalCase per ADR-14).
   Lift copy and microcopy literally.
5. Lift design tokens from [`tokens.css`](./tokens.css) into
   `frontend/tailwind.config.js theme.extend` (Tailwind v3 per ADR-13).

## Tailwind quickstart

```js
// tailwind.config.js (excerpt)
module.exports = {
  content: ['./index.html', './src/**/*.{ts,tsx}'],
  safelist: ['bg-source-nyt', 'bg-source-npr', 'bg-source-grd',
             'text-source-nyt', 'text-source-npr', 'text-source-grd'],
  theme: {
    extend: {
      colors: {
        bg:     { DEFAULT: '#0E0E0E', 2: '#161616', 3: '#1C1C1C', 4: '#232323' },
        ink:    { DEFAULT: '#F2F1EE', 2: '#BDBAB3', 3: '#888581', 4: '#5C5A56' },
        line:   { DEFAULT: '#2A2A2A', 2: '#383838' },
        accent: { DEFAULT: '#FF5C3D', 2: '#FF8467', ink: '#1A0A05' },
        source: { nyt: '#C7B8A8', npr: '#E26A6A', grd: '#6EA8DA' },
        good: '#7CC79A', bad: '#E26A6A', warn: '#F5C26B',
      },
      fontFamily: {
        display: ['Anton', 'Oswald', 'sans-serif'],
        serif:   ['Fraunces', 'Georgia', 'serif'],
        sans:    ['Inter', 'system-ui', 'sans-serif'],
        mono:    ['"JetBrains Mono"', 'ui-monospace', 'monospace'],
      },
      borderRadius: { DEFAULT: '12px', sm: '8px', lg: '18px' },
    },
  },
};
```

The `safelist` entry matters: Tailwind v3 purges class names that aren't
literally present in source. Source-color classes are picked dynamically
(`bg-source-${id}`), so they need explicit safelisting or a static map.

## Components in the design but not in the MVP API contract

Some components in the JSX render features the current API contract doesn't
cover (e.g. `InlineDiff`, `EntitiesCard`, the scrape-run progress modal,
structured chat messages). Treat them as design reference only — do not invent
backend endpoints or schema for them. If pulled into scope, they will land via
a spec change in [`contracts.md`](../../contracts.md) and a new ADR in
[`decisions.md`](../../decisions.md) first.
