## Interfaces / Contracts to Expose

### ORM model ŌĆö `backend/app/models.py` (additions)

```python
class ChatMessage(Base):
    __tablename__ = "chat_messages"
    __table_args__ = (
        sa.CheckConstraint(
            "role IN ('user', 'assistant')",
            name="ck_chat_messages_role",
        ),
        sa.Index(
            "ix_chat_messages_article_id_created_at",
            "article_id",
            "created_at",
        ),
    )

    id: Mapped[int] = mapped_column(sa.Integer, primary_key=True, autoincrement=True)
    article_id: Mapped[int] = mapped_column(
        sa.Integer,
        sa.ForeignKey("articles.id", ondelete="CASCADE"),
        nullable=False,
    )
    role: Mapped[str] = mapped_column(sa.String(20), nullable=False)
    content: Mapped[str] = mapped_column(sa.Text, nullable=False)
    is_error: Mapped[bool] = mapped_column(
        sa.Boolean, nullable=False, server_default=sa.text("false")
    )
    request_id: Mapped[str | None] = mapped_column(sa.String(64), nullable=True)
    created_at: Mapped[datetime] = mapped_column(
        sa.DateTime(timezone=True), nullable=False, server_default=sa.func.now()
    )
```

No back-relationship is added on `Article` in this task (chat is a one-way attach;
the existing `Article.fake` relationship is preserved unchanged).

### Pydantic schemas ŌĆö `backend/app/schemas/chat.py`

```python
class ChatMessageOut(BaseModel):
    model_config = ConfigDict(from_attributes=True)
    id: int
    role: str             # "user" or "assistant" ŌĆö string, not enum
    content: str
    is_error: bool
    request_id: str | None
    created_at: datetime

class ChatHistoryResponse(BaseModel):
    article_id: int
    messages: list[ChatMessageOut]
```

`ChatMessageOut` has exactly the six keys listed (criterion 15). `ChatHistoryResponse`
has exactly two keys (criterion 14). Pydantic `datetime` serialises columns of type
`DateTime(timezone=True)` as ISO 8601 with `+00:00` offset (criterion 18).

### Service function ŌĆö `backend/app/services/chat.py`

```python
async def get_chat_history(
    session: AsyncSession, article_id: int

```

Raises `NotFoundError(f"Article {article_id} not found")` if the article does not exist
(criterion 12). Returns `ChatHistoryResponse` with `messages=[]` for an existing article
that has no messages (criterion 11).

### Router endpoint ŌĆö `backend/app/routers/chat.py`

```text
GET /api/articles/{article_id}/chat ŌåÆ 200 ChatHistoryResponse | 404 {"detail": ...}
```

`article_id` is typed `int` in the path (FastAPI 422 on non-integer paths ŌĆö see open
question 4 in spec, not an acceptance criterion). No query params, no auth, no body.
POST is intentionally not registered (criterion 20).

## Implementation Plan

### 1. Add `ChatMessage` to `backend/app/models.py`

Append the class shown in the contracts section. Keep all imports already at the top
of `models.py` ŌĆö no new imports required.

Verify after writing: `Base.metadata.tables` now includes `chat_messages`, and
`ChatMessage.__table__.indexes` contains the composite index, and the role check
constraint is in `__table_args__`.

### 2. Create the Alembic migration

Run `alembic revision -m "chat messages"` from the `backend/` directory to scaffold a
new revision file. Then hand-edit it to:

- Set `down_revision = "cfe2a836394a"` (criterion 1).
- Implement `upgrade()` with `op.create_table("chat_messages", ...)` containing all
  seven columns from criterion 3, the `ck_chat_messages_role` `CheckConstraint`, and
  the `ForeignKey("articles.id", ondelete="CASCADE")`. Then
  `op.create_index("ix_chat_messages_article_id_created_at", "chat_messages",
  ["article_id", "created_at"])` (criterion 5).
- Implement `downgrade()` with
  `op.drop_index("ix_chat_messages_article_id_created_at", table_name="chat_messages")`
  followed by `op.drop_table("chat_messages")` (criterion 2).

Server defaults must be expressed as SQL primitives so Postgres applies them on
INSERT regardless of whether the ORM provides a value (criterion 6):

- `is_error`: `server_default=sa.text("false")`
- `created_at`: `server_default=sa.func.now()`

`request_id` is declared `nullable=True` with no `server_default` and no unique
constraint (criterion 7, and explicitly out-of-scope per spec).

### 3. Create `backend/app/schemas/chat.py`

Define `ChatMessageOut` and `ChatHistoryResponse` exactly as in the contracts section.

### 4. Create `backend/app/services/chat.py`

Imports: `logging`, `select`, `AsyncSession`, `Article`, `ChatMessage`,
`NotFoundError`, `ChatMessageOut`, `ChatHistoryResponse`.

```python
log = logging.getLogger(__name__)

async def get_chat_history(
    session: AsyncSession, article_id: int
) -> ChatHistoryResponse:
    article_exists = await session.scalar(
        select(Article.id).where(Article.id == article_id)
    )
    if article_exists is None:
        raise NotFoundError(f"Article {article_id} not found")

    result = await session.execute(
        select(ChatMessage)
        .where(ChatMessage.article_id == article_id)
        .order_by(ChatMessage.created_at.asc(), ChatMessage.id.asc())
    )
    rows = result.scalars().all()

    messages = [ChatMessageOut.model_validate(row) for row in rows]
    log.info(
        "chat.history.get article_id=%d count=%d", article_id, len(messages)
    )
    return ChatHistoryResponse(article_id=article_id, messages=messages)
```

The `ORDER BY created_at ASC, id ASC` clause delivers chronological ordering with the
stable tie-break (criteria 10, 13). The empty-history case naturally produces
`messages=[]` because `rows` is empty (criterion 11). The log line fires for both
populated and empty cases ŌĆö see open question 6 in spec.

### 5. Create `backend/app/routers/chat.py`

```python
from fastapi import APIRouter, Depends
from sqlalchemy.ext.asyncio import AsyncSession

from app.db import get_session
from app.schemas.chat import ChatHistoryResponse
from app.services import chat as chat_service

router = APIRouter(prefix="/api")


@router.get(
    "/articles/{article_id}/chat", response_model=ChatHistoryResponse
)
async def get_chat_history(
    article_id: int,
    session: AsyncSession = Depends(get_session),
) -> ChatHistoryResponse:
    return await chat_service.get_chat_history(session, article_id)
```

`prefix="/api"` plus the path `"/articles/{article_id}/chat"` produces the full URL
`/api/articles/{article_id}/chat` (criterion 9). No POST handler is declared, which
gives FastAPI's automatic 405 (criterion 20). No auth dependency (criterion 19).
