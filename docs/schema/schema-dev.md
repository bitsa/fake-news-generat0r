## Implementation plan

Steps are ordered so imports resolve cleanly at each stage. Each step must
compile before the next begins.

### Step 1 — Create `backend/app/sources.py`

Define `Source(StrEnum)` with members `NYT`, `NPR`, `GUARDIAN` in that order.
Define `FEED_URLS: dict[Source, str]` with the three exact URLs above.
No other exports; no class methods.

### Step 2 — Create `backend/app/models.py`

- Import: `datetime`, `sqlalchemy as sa`, `sqlalchemy.orm.{DeclarativeBase, Mapped, mapped_column, relationship}`, `app.sources.Source`.
- Declare `Base(DeclarativeBase)`.
- **`Article`**:
  - `id`: `mapped_column(sa.Integer, primary_key=True, autoincrement=True)`
  - `source`: `mapped_column(sa.Enum(Source, name="source_type", values_callable=lambda e: [m.value for m in e]), nullable=False)` — `values_callable` ensures Postgres labels are `.value` strings, not Python member names.
  - `title`: `mapped_column(sa.Text, nullable=False)`
  - `description`: `mapped_column(sa.Text, nullable=True)`
  - `url`: `mapped_column(sa.Text, nullable=False, unique=True)`
  - `published_at`: `mapped_column(sa.DateTime(timezone=True), nullable=True)`
  - `created_at`: `mapped_column(sa.DateTime(timezone=True), nullable=False, server_default=sa.func.now())`
  - `fake`: `relationship("ArticleFake", back_populates="article", uselist=False, cascade="all, delete-orphan")`
- **`ArticleFake`**:
  - `article_id`: `mapped_column(sa.Integer, sa.ForeignKey("articles.id", ondelete="CASCADE"), primary_key=True)` — single column simultaneously PK and FK; enforces 1:1 by construction.
  - `transform_status`: `mapped_column(sa.String(20), nullable=False, server_default="pending")`
  - `__table_args__`: include `sa.CheckConstraint("transform_status IN ('pending', 'completed')", name="ck_article_fakes_transform_status")`
  - `title`: `mapped_column(sa.Text, nullable=True)`
  - `description`: `mapped_column(sa.Text, nullable=True)`
  - `model`: `mapped_column(sa.String(100), nullable=True)`
  - `temperature`: `mapped_column(sa.Double, nullable=True)`
  - `created_at`: `mapped_column(sa.DateTime(timezone=True), nullable=False, server_default=sa.func.now())`
  - `article`: `relationship("Article", back_populates="fake")`

### Step 3 — Create the genesis migration

Run from `backend/`:

```bash
alembic revision -m "initial schema"
```

This generates `backend/migrations/versions/<12-char-hex>_initial_schema.py`.
Replace the body by hand — do not use `--autogenerate` (hand-written per context.md decisions).

At the top of the generated file, add:

```python
from app.sources import Source
from sqlalchemy.dialects import postgresql
```

In `upgrade()`:

1. Build and create the enum:

   ```python
   source_type = postgresql.ENUM(*[s.value for s in Source], name="source_type", create_type=False)
   source_type.create(op.get_bind(), checkfirst=False)
   ```

   `create_type=False` prevents SQLAlchemy from issuing `CREATE TYPE` inside `create_table` — the `source_type.create()` call owns the lifecycle.
2. `op.create_table("articles", ...)` with all columns matching `docs/schema.md`. The `source` column uses the `source_type` object above so SQLAlchemy emits `source source_type`, not a re-creation. Include `sa.UniqueConstraint("url", name="uq_articles_url")`.
3. `op.create_index("ix_articles_source", "articles", ["source"])`
4. `op.create_index("ix_articles_published_at", "articles", [sa.text("published_at DESC NULLS LAST")])` — expression index; cannot be expressed as a simple column list.
5. `op.create_table("article_fakes", ...)` with all columns. Include the `CHECK (transform_status IN ('pending', 'completed'))` as `sa.CheckConstraint(...)` inside `create_table`. No additional indexes — `article_id` PK provides its own index.

In `downgrade()` (reverse order):

1. `op.drop_table("article_fakes")`
2. `op.drop_index("ix_articles_published_at", table_name="articles")`
3. `op.drop_index("ix_articles_source", table_name="articles")`
4. `op.drop_table("articles")`
5. `source_type.drop(op.get_bind(), checkfirst=False)`

### Step 4 — Add migration to lifespan startup

In `backend/app/main.py`, before the existing `yield`:

```python
import asyncio
from alembic.config import Config
from alembic.command import upgrade as alembic_upgrade

async def _run_migrations() -> None:
    cfg = Config("alembic.ini")
    await asyncio.to_thread(alembic_upgrade, cfg, "head")
```

Call `await _run_migrations()` at the start of the `lifespan` context (before `yield`). `asyncio.to_thread` is required because `alembic.command.upgrade` is synchronous and must not block the event loop.

The app is always run from the `backend/` directory (local) or `/app` (Docker container), so `Config("alembic.ini")` resolves correctly in both contexts.

## Unit tests required

Tests live under `backend/tests/unit/`. They cover behaviors that do not
require a live Postgres — integration-level criteria (enum constraint
rejection, cascade delete, migration round-trip) belong to the QA pass.

### `tests/unit/test_sources.py`

- `test_source_members_exact` — `set(Source) == {Source.NYT, Source.NPR, Source.GUARDIAN}` and `len(Source) == 3`
- `test_source_member_values` — `.value` of each member equals `"NYT"`, `"NPR"`, `"Guardian"` exactly (capitalization matters — `"Guardian"` not `"GUARDIAN"`)
- `test_source_is_strenum` — `issubclass(Source, str)` is True; a `Source` member equals its value string (`Source.NYT == "NYT"`)
- `test_source_declaration_order` — `list(Source)` is `[Source.NYT, Source.NPR, Source.GUARDIAN]` in that order (migration enum label order depends on it)
- `test_feed_urls_keys_match_source` — `set(FEED_URLS) == set(Source)`
- `test_feed_urls_has_exactly_three_entries` — `len(FEED_URLS) == 3`
- `test_feed_urls_values_exact` — each URL matches the spec's literal string exactly; assert `url.startswith("https://")`

### `tests/unit/test_models.py`

- `test_tablenames` — `Article.__tablename__ == "articles"`, `ArticleFake.__tablename__ == "article_fakes"`
- `test_metadata_tables` — `set(Base.metadata.tables) == {"articles", "article_fakes"}` — guards against accidental extra tables
- `test_article_source_enum_binding` — inspect `Article.__table__.c.source.type`; assert it is a `sa.Enum` whose `name == "source_type"` and whose persisted labels equal `[m.value for m in Source]`
- `test_article_description_nullable` — `Article.__table__.c.description.nullable is True`
- `test_article_url_unique` — `Article.__table__.c.url.unique is True` or a `UniqueConstraint` on `url` exists in `Article.__table__.constraints`
- `test_article_fake_pk_is_fk` — inspect `ArticleFake.__table__.c.article_id`: `primary_key is True`, has exactly one `ForeignKey` whose target table is `articles` and target column is `id`, and whose `ondelete == "CASCADE"`
- `test_article_fake_transform_status_check` — a `CheckConstraint` exists in `ArticleFake.__table__.constraints` whose SQL text includes both `'pending'` and `'completed'`
- `test_article_fake_nullable_columns` — `title`, `description`, `model`, `temperature` columns on `ArticleFake` are all `nullable is True`
- `test_article_fake_no_extra_indexes` — `len(ArticleFake.__table__.indexes) == 0` (implicit PK index is not in `.indexes`)
- `test_article_fake_relationship` — `Article.fake` attribute exists and its `property.mapper.class_` is `ArticleFake`
- `test_article_back_ref` — `ArticleFake.article` attribute exists and its `property.mapper.class_` is `Article`

### `tests/unit/test_migration_module.py`

Load the migration via `importlib` by globbing `backend/migrations/versions/*.py`
(excluding `.gitkeep` and `__pycache__`):

- `test_single_revision_file` — exactly one `.py` revision file under `migrations/versions/`
- `test_down_revision_is_none` — `module.down_revision is None`
- `test_revision_is_string` — `isinstance(module.revision, str)` and non-empty
- `test_migration_imports_source` — migration source text contains the literal `from app.sources import Source`
- `test_migration_derives_labels_from_source` — migration source does **not** contain a hardcoded string list with all three labels as adjacent literals (e.g. `"NYT", "NPR", "Guardian"` in sequence); labels must be derived from `Source` at migration time
- `test_upgrade_downgrade_callable` — `module.upgrade` and `module.downgrade` are callable (do not call them — they require a live DB and Alembic context)

### 2. `chat_messages` out of scope

`docs/schema-spec.md` explicitly excludes `chat_messages`. The models file
will only contain `Article` and `ArticleFake`. When the `chat_messages`
task is implemented, it will extend `models.py` (or its replacement).

### 3. Startup migration in tests

The `conftest.py` fixture creates a `FastAPI` app instance, which triggers
the lifespan `alembic upgrade head` call. Unit tests that import `app.main`
will attempt to run migrations against the configured `DATABASE_URL`. The
existing conftest sets `DATABASE_URL` to a test Postgres instance before
imports. For pure unit tests that don't start the app this is not an issue,
but any test that instantiates `app` via the `app` fixture will run
migrations. This is the expected behavior for integration fixtures;
unit-only tests should not use the `app` fixture.
